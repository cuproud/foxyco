# GigPilot MVP — Project Specification & Development Prompt

You are a senior Android engineer, software architect, UI/UX designer, and product manager.

Your goal is to help me build a production-quality Android application from scratch.

Do NOT rush into writing code.

Think like a senior software engineer designing an application that will continue growing for years.

---

# Project Name

GigPilot (working title)

---

# Project Goal

GigPilot is a completely free Android application for gig economy drivers.

The MVP has ONE purpose only:

**Instantly analyze an incoming delivery or ride offer and tell the driver whether it is a GOOD, AVERAGE, or BAD offer.**

The app must work completely locally.

No cloud.

No backend.

No subscriptions.

No login.

No analytics.

No AI chat.

No unnecessary features.

Only solve one problem extremely well.

---

# Product Philosophy

I do NOT want to build another feature-heavy application.

I want to build the best offer analyzer possible.

Every design and engineering decision should prioritize:

* simplicity
* speed
* reliability
* maintainability
* modularity
* clean architecture
* future scalability

The app should feel polished and professional despite having very few features.

---

# MVP Scope

The application should only do the following:

1. Detect an incoming offer.
2. Read the offer information.
3. Calculate profitability.
4. Classify the offer.
5. Display the result instantly.

Nothing more.

# Performance Requirements

The application should feel instant.

Target:

Offer detected

↓

Information parsed

↓

Calculation completed

↓

Recommendation displayed

within approximately 300 milliseconds whenever feasible.

Battery consumption should be minimal.

Memory usage should remain low.

No unnecessary background work.

---

# Core Workflow

Incoming Offer

↓

Accessibility Service detects the screen or notification.

↓

Offer Parser extracts:

* payout
* pickup distance
* delivery distance
* total distance
* estimated duration (if available)

↓

Profit Calculator computes:

* fuel cost
* vehicle wear
* estimated profit
* dollars per kilometer
* dollars per hour (when time is available)

↓

Decision Engine determines:

GOOD

AVERAGE

BAD

↓

Overlay immediately displays recommendation.

---

# Initial Supported Platforms

Design the architecture so support for multiple delivery platforms can be added later.

For the MVP, the parser should be modular and independent of platform-specific UI so new parsers can be introduced without changing the business logic.

---

# Driver Settings

The app should allow the user to configure:

Vehicle:

* fuel economy
* fuel type

Operating Costs:

* fuel price
* maintenance cost per kilometer
* depreciation cost per kilometer (optional)

Offer Filters:

* minimum payout
* minimum dollars per kilometer
* minimum hourly earnings
* maximum delivery distance

Keep settings minimal and easy to understand.

# Offer Evaluation

Good

Poor

The decision should be based on configurable rules rather than hardcoded values.
